# Counter App - 设计规格说明书

## 1. 项目概述

### 1.1 产品定位
一款安卓端智能计数应用，通过摄像头捕获重复动作、麦克风捕获重复声音进行自动计数。支持运动健身场景（内置模板）和通用自定义场景（用户录制样本）。

### 1.2 核心功能
- 摄像头动作检测计数（视觉）
- 麦克风声音检测计数（音频）
- 用户自定义样本录制与模板匹配
- 样本匹配容差（阈值）调节
- 多用户数据管理
- 计数历史记录与数据分析
- 本地数据导出/导入备份
- 深色模式设置（跟随系统/浅色/深色）
- 生物识别快速切换用户（指纹/面部识别）
- 静态姿势计时（Phase 2：平板支撑、扎马步、倒立等）

### 1.3 目标用户
- 健身爱好者（俯卧撑、深蹲、跳绳等）
- 需要通用计数工具的用户

### 2.2 设计原则
1. **引擎解耦**：视觉引擎和音频引擎完全独立，各自有独立的生命周期和状态管理
2. **可插拔算法**：传统算法和 TFLite 模型通过统一接口实现，运行时可选
3. **离线优先**：所有数据处理在本地完成，无需联网
4. **多用户隔离**：每个用户有独立的数据库和文件空间

## 3. 模块设计

### 3.1 视觉检测引擎（VisionDetectionEngine）

**职责**：从摄像头帧流中识别周期性动作，输出计数事件。

**核心流程**：
1. **帧流预处理**：CameraX → YUV → RGB 转换，可选降低分辨率以提升性能
2. **运动检测**：帧差分/光流 → 判断是否有显著运动，若无运动跳过本帧
3. **周期性分析**：分析运动强度序列的周期性，计算自相关函数 → 找峰值间隔
4. **模板匹配（可选）**：若用户录制了样本，计算相似度；若相似度 > 阈值，判定为一次有效动作
5. **防抖动/去重**：最小间隔过滤（如 500ms 内不重复计数），置信度加权

**接口定义**：
```kotlin
interface DetectionEngine {
    fun start(config: DetectionConfig)
    fun pause()
    fun resume()
    fun stop()
    fun setTemplate(template: Template?)
    fun setThreshold(threshold: Float)
    val countEvents: Flow<CountEvent>
}
```

### 3.2 音频检测引擎（AudioDetectionEngine）

**职责**：从麦克风音频流中识别周期性声音，输出计数事件。

**核心流程**：
1. **音频采集**：AudioRecord 实时采集 PCM 数据
2. **频谱分析**：FFT 转换 → 频带能量计算
3. **能量变化检测**：检测能量突增/突减事件
4. **周期性分析**：分析能量变化序列的周期性
5. **模板匹配（可选）**：与视觉引擎共用模板匹配逻辑
6. **去重输出**：最小间隔过滤，输出计数事件

### 3.3 模板匹配引擎（TemplateMatchingEngine）

**职责**：计算实时输入与用户样本的相似度，决定是否触发计数。

**流程**：
1. **特征提取（录制时）**：
   - 视觉样本：提取光流直方图 / 关键帧特征向量
   - 音频样本：提取 MFCC（梅尔频率倒谱系数）特征序列
2. **特征存储**：特征向量 + 原始媒体文件路径 + 阈值
3. **相似度计算（运行时）**：
   - 视觉：当前帧特征与样本特征计算余弦相似度 / 欧氏距离
   - 音频：当前音频段特征与样本特征计算 DTW（动态时间规整）距离
4. **阈值判断**：相似度 > 阈值 → 触发计数

### 3.4 计数服务（CountingService）

**职责**：管理检测会话的生命周期，聚合检测引擎结果，持久化数据。

**状态机**：
```
[IDLE] --开始计数--> [RUNNING] --暂停--> [PAUSED]
  ^                       | --完成--> [COMPLETED]
  |                       | --取消--> [CANCELLED]
  |                       |
[COMPLETED/CANCELLED] <--完成/取消后重置--
```

### 3.5 数据分析模块

**职责**：对历史计数数据进行统计分析和可视化。

**功能**：
- 单次会话分析：用时、次数、频率（次/分钟）
- 趋势分析：周/月/年计数趋势
- 对比分析：不同模板的使用频率对比

## 4. 数据模型

### 4.1 用户（User）
```kotlin
@Entity
data class User(
    @PrimaryKey val id: Long = 0,
    val name: String,
    val createdAt: Long = System.currentTimeMillis(),
    val avatarPath: String? = null
)
```

### 4.2 计数会话（CountingSession）
```kotlin
@Entity
data class CountingSession(
    @PrimaryKey val id: Long = 0,
    val userId: Long,
    val name: String,           // 如"俯卧撑训练"
    val templateId: Long?,        // 关联模板（内置或自定义）
    val sensorType: SensorType,   // VISION / AUDIO
    val startTime: Long,
    val endTime: Long? = null,
    val targetCount: Int? = null, // 目标次数
    val finalCount: Int = 0,     // 实际完成次数
    val status: SessionStatus = SessionStatus.RUNNING
)

enum class SensorType { VISION, AUDIO }
enum class SessionStatus { RUNNING, PAUSED, COMPLETED, CANCELLED }
```

### 4.3 计数事件（CountEvent）
```kotlin
@Entity
data class CountEvent(
    @PrimaryKey val id: Long = 0,
    val sessionId: Long,
    val timestamp: Long,
    val count: Int,           // 当前累计次数
    val confidence: Float     // 置信度 (0.0 ~ 1.0)
)
```

### 4.4 模板（Template）
```kotlin
@Entity
data class Template(
    @PrimaryKey val id: Long = 0,
    val userId: Long?,        // null 表示内置模板
    val name: String,
    val type: TemplateType,   // BUILTIN / CUSTOM
    val sensorType: SensorType,
    val mediaPath: String,    // 样本媒体文件路径
    val featureVector: ByteArray, // 提取的特征向量
    val threshold: Float = 0.7f,  // 默认匹配阈值
    val createdAt: Long = System.currentTimeMillis()
)

enum class TemplateType { BUILTIN, CUSTOM }
```

### 4.5 数据库关系
```
User 1--N CountingSession
User 1--N Template
CountingSession 1--N CountEvent
Template (可选) --1 CountingSession
```

## 5. UI/UX 设计

### 5.1 核心界面结构

```
┌─────────────────────────────────────┐
│  顶部：用户切换 + 当前时间           │
├─────────────────────────────────────┤
│                                     │
│  中部：大号计数显示 (如 0 / 50)      │
│        进度环/进度条                 │
│                                     │
├─────────────────────────────────────┤
│  底部操作区：                         │
│  [选择模板] [开始/暂停] [结束保存]   │
│                                     │
├─────────────────────────────────────┤
│  传感器选择：                         │
│  [摄像头] [麦克风] (开关切换)        │
│                                     │
├─────────────────────────────────────┤
│  Tab栏：计数 | 历史 | 分析 | 设置   │
└─────────────────────────────────────┘
```

### 5.2 关键交互流程

**开始计数**：
1. 选择用户 → 选择模板 → 选择传感器类型 → 点击开始
2. 实时显示计数和摄像头预览（或音频波形）
3. 可暂停/继续，或结束保存

**录制样本**：
1. 进入"样本管理" → 点击"录制样本"
2. 选择传感器类型 → 开始录制 → 执行动作/发出声音
3. 系统自动提取特征 → 预览并调节阈值
4. 命名并保存

**查看历史与分析**：
1. 切换到"历史"Tab → 按时间线查看会话列表
2. 点击进入详情：次数、用时、频率、趋势图
3. 切换到"分析"Tab：周/月/年趋势、各模板使用统计

**切换用户**：
1. 点击顶部用户切换按钮，或在设置中选择"切换用户"
2. 系统检测是否支持生物识别（指纹/面部）
3. 若支持，弹出生物识别验证 → 验证通过后切换到对应用户
4. 若不支持，手动输入用户名和密码切换

**设置深色模式**：
1. 进入"设置"Tab → 选择"外观"
2. 选择深色模式选项：跟随系统 / 浅色 / 深色
3. 切换后立即生效

## 6. 技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **语言** | Kotlin | 现代、安全、与 Jetpack 深度整合 |
| **UI** | Jetpack Compose | 声明式 UI，开发效率高 |
| **架构** | MVVM + Repository + Clean Architecture | 结构清晰，易于测试 |
| **依赖注入** | Hilt | 简化依赖管理 |
| **异步处理** | Kotlin Coroutines + Flow | 响应式数据流 |
| **数据库** | Room + SQLite | 类型安全、支持协程 |
| **图表** | Compose Charts / MPAndroidChart | 数据可视化 |
| **相机** | CameraX | Google 官方推荐，API 稳定 |
| **音频** | AudioRecord | 低延迟音频采集 |
| **图像处理** | OpenCV Android | 帧差分、光流等 |
| **可选 ML** | TensorFlow Lite | 复杂动作识别（Phase 3） |

## 7. 实现阶段规划

### Phase 1：MVP（预计 4-6 周）
**目标**：可运行的基础计数器
- 单用户硬编码（Phase 2 再做多用户）
- 仅摄像头视觉检测（传统算法：帧差分）
- 内置 3-5 个简单模板（拍手、挥手、深蹲等）
- 基础计数 + 会话保存 + 历史列表
- 数据导出（JSON 格式）

### Phase 2：完整功能（预计 3-4 周）
**目标**：功能完整的多用户计数应用
- 多用户支持（切换、创建、删除）
- 音频检测引擎
- 自定义样本录制与模板匹配
- 容差（阈值）调节 UI
- 数据分析与图表（趋势、统计）
- 数据导入/导出备份

### Phase 3：增强优化（预计 2-3 周）
**目标**：体验优化与扩展性
- TFLite 可选模型（复杂动作识别）
- 远程备份接口预留（网盘/NAS）
- UI 美化与过渡动画
- 性能优化（低端设备适配、电池优化）
- 单元测试覆盖

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 复杂场景下检测准确率不足 | 高 | 提供阈值调节；Phase 3 引入 TFLite |
| 低端设备性能瓶颈 | 中 | 提供分辨率/帧率降级选项；OpenCV 轻量配置 |
| 音频检测受环境噪音干扰 | 中 | 频带过滤；用户可录制排除噪音的样本 |
| 多用户数据隔离漏洞 | 中 | Repository 层强制 userId 过滤；单元测试覆盖 |
| 样本录制体验不佳 | 低 | 录制时实时预览特征提取效果；提供重录选项 |
